# Interactive Renewable Energy Dashboard

An interactive Plotly/Dash web application for exploring renewable energy scenarios, battery storage optimization, and comparative metrics for Germany's electricity system.

## Features

### 📊 Overview Dashboard

- **KPI Cards**: Average coverage, shortfall days, average shortfall, peak shortfall
- **Scenario Selector**: Switch between worst, average, and best case scenarios
- **Metrics Comparison**: Bar charts comparing all scenarios across key metrics

### 📈 Time Series Explorer

- **Interactive Charts**: Zoom, pan, and hover for detailed values
- **Date Range Slider**: Filter data to specific time periods
- **Series Toggles**: Show/hide wind, solar, and demand independently
- **Shortfall Analysis**: Dedicated visualization of daily unmet demand

### 🔋 Battery Storage Analysis

- **What-If Analysis**: Explore 4 battery capacity configurations (5/10/20/50 GWh)
- **State of Charge**: Visualize battery SOC over time
- **Dispatch Patterns**: See charging/discharging behavior
- **Capacity Comparison**: Analyze shortfall reduction vs. battery size
- **Economic Metrics**: CAPEX, O&M costs, payback periods

### 📋 Comprehensive Metrics

- **Metrics Table**: Side-by-side scenario comparison
- **Seasonal Patterns**: Identify when shortfalls are most severe
- **Coverage Distribution**: Histogram of hourly coverage across scenarios
- **Battery Economics**: Cost comparison across scenarios and battery sizes

## Quick Start

### Prerequisites

- Python 3.8+
- Required packages (see requirements.txt)

### Installation

1. **Install dependencies** (if not already done):

```bash
pip install -r requirements.txt
```

2. **Run the dashboard**:

```bash
python dashboard.py
```

3. **Open in browser**:
Navigate to `http://localhost:8050`

## Interpretation Guide

### Coverage %

- Percentage of electricity demand met by renewables
- Worst case: ~22% average
- Average case: ~29% average
- Best case: ~39% average

### Shortfall (GW)

- Daily unmet electricity demand
- Higher in winter, lower in summer
- Key input for battery storage sizing

### Battery Reduction

- Percentage of shortfall eliminated by storage
- 5 GWh battery: ~3-6% reduction
- 50 GWh battery: ~11-14% reduction
- Larger capacity needed for near-complete coverage

### SOC Swing

- Difference between battery's maximum and minimum charge during period
- Indicates cycling intensity
- Higher swings = more active trading

## Performance Notes

## Troubleshooting

### Dashboard won't start

```bash
# Check Python version
python --version  # Should be 3.8+

# Verify all packages installed
pip install -r requirements.txt

# Check if port 8050 is already in use
# If so, change port in dashboard.py line: app.run_server(port=8051)
```

### Data not loading

- Verify all CSV files are in the same directory
- Check file names match exactly (case-sensitive on Linux/Mac)
- Ensure scenario_forecasts.csv has correct columns

### Charts not displaying

- Clear browser cache
- Try a different browser
- Check browser console for errors (F12 Developer Tools)

## Extending the Dashboard

### Add New Metrics

Edit the relevant callback function and add a new card or chart section.

### Change Color Scheme

Modify the color constants at the top of dashboard.py:

```python
COLOR_WORST = '#e08adf'
COLOR_AVERAGE = '#c06bb7'
COLOR_BEST = '#9b59b6'
```

### Add New Pages

1. Create a new function like `create_custom_page()`
2. Add route in `display_page()` callback
3. Add navigation link in navbar

### Export Data

Modify dashboard to add download buttons using `dcc.Download` component.
